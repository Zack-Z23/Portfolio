SSL

Many people have wondered, what is SSL? SSL typically stands for Secure Sockets Layer; this protocol allows you to have secure transactions on your web server but it can also be used to put your bank details into it. Every time you pay for something or put sensitive data on a website like amazon, best buy, and Newegg. All those websites have used the protocols like SSL or TLS. TLS stands for Transport Layer Security and what it does is a better and improved version of SSL which is why most companies with websites would have that protocol and that it would be more likely that websites like amazon and best buy would use it. Just like TLS, the SSL certificate authority verifies a website and gives it a digital certification so users and computers understand that the website they are going into is secure.

If you wanna know if your website is secure, then your website address should start with HTTPS(Hypertext Transfer Protocol Secure) or HTTP(Hypertext Transfer Protocol) which means that your website has a TLS or SSL certification.

That is why HTTP or HTTPS matters so much when you are buying something online or logging into your bank, those two prefixes are a way to tell you that your sensitive data is secured to a certain extent and that it has been encrypted. To secure your website you should also consider encrypting your website cause that is a major part of keeping your website secure. Encryption is taking your data and turning it into a hidden message only the webserver would understand, this encryption allows you to keep your sensitive data secure and people trying to access it away. Websites don’t always have to be secure. Some websites don’t have a login/sign up so in those cases you won’t have a very secure website. Finding a website that isn’t secured at all because either the creator forgot to make it secure or just wasn't necessary is a rare instance but some websites aren’t secured for a reason, that reason can range from taking your information to using that website to edit it or is still in the progression of getting things added to it. Although there is information, like your bank details, that you need to have a secure website to put that type of information into that website.

In the end, being secure means having the trust of the user, using that website, to put sensitive data into it. Having a secure website not only is a benefit to the users but a benefit to the creator of the website. If a user likes the website just because of how secure and how well the data is handled, the creator of that website can be offered a job in a big company that allows him to gain more money and help more users on a wider scale. So having a secure website not only is a benefit to the users who are using it but also a benefit to the creator.
